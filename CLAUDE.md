# プロジェクト概要: Google Drive統合型 Markdownエディタの開発

あなたはシニアフロントエンドエンジニアとして、以下の要件に基づきWebアプリケーションの開発を支援してください。推測や不確実な技術スタックの導入は避け、指示された制約とアーキテクチャに厳密に従ってコードを生成・提案してください。

## 1. プロジェクトの目的（最優先事項）
Google Drive上の Markdownファイル（.md）をブラウザ上でシームレスに閲覧・編集・上書き保存できる独立したWebアプリケーションを構築する。
ユーザーがGoogle DriveのUI上でファイルをダブルクリック（または「アプリで開く」を選択）した際、本アプリが起動し、直接編集できるUXを提供する。

## 2. 技術スタックとインフラ制約
*   **フロントエンドフレームワーク:** React (Vite環境) + TypeScript
*   **ホスティング:** Cloudflare Pages (静的サイトとしてデプロイ。Drive UI IntegrationのためHTTPSの静的URLを想定)
*   **テキストエディタエンジン:** CodeMirror 6
*   **Markdownプレビュー:** react-markdown
*   **データフェッチ・状態管理:** TanStack Query (React Query) または SWR （Google Drive APIとの非同期通信、ローディング状態、エラーハンドリングを厳格に管理するため）
*   **認証・認可:** Google Identity Services (GIS) / OAuth 2.0
*   **要求スコープ:** `https://www.googleapis.com/auth/drive.file` （アプリで作成・開いたファイルのみに限定）

## 3. 開発ロードマップと現在のフェーズ
本プロジェクトはリスクを最小化するため、段階的に実装を行います。現在は**【フェーズ1】**の着手段階です。フェーズ2以降の機能は考慮しつつも、まずはフェーズ1の完遂に集中してください。

*   **【現在のタスク】フェーズ1: Driveからの起動と閲覧（Read-Only）**
    *   Drive UI Integrationから渡されるURLパラメータ（`?state=...`）のパースとファイルIDの抽出。
    *   GISを用いたOAuth 2.0認証フローの実装（アクセストークンの取得と管理）。
    *   Google Drive API (`v3/files/{fileId}?alt=media`) を用いたテキストデータの取得。
    *   `react-markdown` を用いたプレビュー表示。
*   **【将来のタスク】フェーズ2: 編集と上書き保存（Update）**
    *   CodeMirror 6の統合。
    *   Drive API (`PATCH`) による上書き保存（リビジョン維持）。
*   **【将来のタスク】フェーズ3: 高度なUX**
    *   Editor/Previewの分割表示と、AST（ソースマッピング）を用いた高度なスクロール同期。

## 4. フェーズ1における具体的な実装要求（指示）
以下のステップに従って、必要なファイル構成と具体的なTypeScript/Reactのコードを提示してください。

1.  **URLパラメータの解析ロジック:**
    Google Driveからリダイレクトされた際のURLに含まれる `state` パラメータ（JSON文字列がURLエンコードされたもの）から、`action` が `"open"` であることを確認し、`ids` 配列から最初のファイルIDを安全に取り出すユーティリティ関数を作成してください。
2.  **認証フローの実装方針:**
    Google Identity Services (GIS) を用いて、Drive APIを叩くためのアクセストークンを取得するReactフックを作成してください。ユーザーに認証を促すUI（ログインボタン等）も含めてください。
3.  **データフェッチとエラーハンドリング:**
    ファイルIDとアクセストークンを用いてDrive APIからファイル内容（プレーンテキスト）を非同期で取得する処理を、TanStack Query (またはSWR) を用いて実装してください。ネットワークエラーや権限エラー（401, 403, 404）に対するフォールバックUIも考慮してください。
